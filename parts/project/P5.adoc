=== (P.5) Required technology elements

ifdef::env-draft[]
TIP: _External systems, hardware and software, expected to be necessary for building the system. Although the actual use of such products belongs to design and implementation rather than requirements, it is part of the requirements task to identify elements whose availability is critical to the success of the project — an important element of risk analysis (<<p6>>)._  <<BM22>>
endif::[]

The following external technology elements are required for the successful development and deployment of the Sanskrit Cipher system:

==== Programming Languages
- **Python** for machine learning models and the ML pipeline.
- **JavaScript/TypeScript** for the Electron desktop app.

==== Libraries & Frameworks
- **OpenCV** for image processing.
- **PyTorch** for machine learning models.
- **You Only Look Once (YOLO)** models (Ultralytics) for segmentation and line detection.
- **EasyOCR** for scale detection from rulers.
- **React** and **Electron** for the desktop UI.
- **Vite** for frontend tooling.
- **Cascading Style Sheets (CSS)** via **Tailwind** for styling.
- **SQLite** with a Node binding for local persistence.

==== Pre-trained Models
- YOLO-based segmentation and line detection models.
- Convolutional Neural Network (CNN)-based line count and script type classifiers.

==== Development Tools
- **VSCode** as the main Integrated Development Environment (IDE).
- **LaTeX/Markdown** for documentation.

==== Testing & Code Quality Tools
- **Pytest** for Python tests.
- **ESLint** for frontend linting.

==== Build & Automation
- **Makefiles** for document builds.
- **GitHub Actions** for continuous integration/continuous delivery (CI/CD).

==== Deployment & Infrastructure
- **GitHub Projects** for version control and project management.
- **SQLite** as the local database backend.
