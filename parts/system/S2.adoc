== S.2.1 Front End

=== Functional Properties:

*Fragment Search & Browsing*: The system will provide a home screen with project history and fragment ID search (including autocomplete) to quickly locate fragments.

*Filtering*: The system will allow users to filter fragments by line count range, script type, edge flags, circle detection, and user-defined custom fields.

*Canvas Manipulation*: The system will support drag-and-drop placement of fragments onto a canvas with move, resize, rotate, multi-select, lock/unlock, grid toggle, and segmented/original view toggle.

*Metadata Viewing & Editing*: The system will display fragment metadata and allow users to edit fields such as line count, script type, edge flags, circle detection, scale data, and custom fields.

*Project Management*: The system will allow users to create, rename, delete, auto-save, and restore reconstruction projects.

*Fragment Upload*: The system will support importing new fragment images into the local database.

=== Non-Functional Properties:

*Usability*: The front-end interface will be designed to be intuitive and user-friendly, ensuring that users can easily navigate and utilize the system's features without extensive training.

*Performance*: The front-end will be optimized for responsiveness during canvas interactions and while browsing large fragment collections via pagination.

== S.2.2 Back End

=== Functional Properties:

*Electron IPC Interface*: The back-end will provide a local IPC interface between the Electron main process and the UI for fragment, project, file, and custom-filter operations.

*Fragment Data Access*: The back-end will query, search, and update fragment metadata in SQLite.

*Project Persistence*: The back-end will save and load canvas state for projects.

*File Import & Storage*: The back-end will handle image selection, import, and storage, including a cache for segmented images.

*Custom Filters*: The back-end will allow creation, update, and deletion of custom metadata fields used for filtering.

*Edge Matching*: The back-end will compare tear-edge descriptors to generate candidate matches.

=== Non-Functional Properties:

*Reliability*: The back-end will include error handling and maintain database integrity for local operations.

*Portability*: The back-end will run locally as part of the desktop application without requiring external services.

== S.2.3 AI/ML Cluster

=== Functional Properties:

*Segmentation*: The system will segment fragments, store contour coordinates, and generate transparent PNGs for UI display.

*Line Count Classification*: The system will predict the number of text lines in a fragment.

*Line Detection*: The system will detect individual text lines and store bounding boxes and counts.

*Script Type Classification*: The system will classify script type labels with confidence scores.

*Edge Detection*: The system will infer border edges (top, bottom, left, right) and edge-piece flags based on contours.

*Circle Detection*: The system will detect circular marks and store a boolean flag.

*Scale Detection*: The system will estimate pixels-per-unit for fragments that include rulers.


=== Non-Functional Properties:

*Configurability*: The pipeline will be configuration-driven, with model versions recorded in the database for traceability.

*Efficiency*: The pipeline will support batch processing with resume capability and optional dry-run mode.

*Maintainability*: The ML pipeline will use a plugin architecture to add or update processors.
