=== (S.6) Verification and acceptance criteria

==== Overview

This section specifies conditions under which the Sanskrit Manuscript Fragment Reconstruction Platform will be deemed satisfactory for local desktop research use. The V&V framework focuses on the Electron app, SQLite data store, and the offline ML pipeline.

==== 1. Functional Verification Criteria

*Fragment Search and Filtering*

* **Search and Filter Accuracy**: Fragment ID search and metadata filters return correct results. Success threshold: queries complete within 3 seconds on the local dataset and return expected counts for known test cases.

*Canvas Workspace*

* **Drag/Drop and Manipulation**: Fragments can be placed, moved, resized, rotated, multi-selected, locked, and deleted without errors. Success threshold: interactions remain responsive (<=200ms feedback) for typical canvas sizes.

*Project Persistence*

* **Save/Restore**: Auto-save and manual save persist canvas state; loading a project restores fragment positions, transforms, and locks. Success threshold: 100% fidelity on a test project.

*Uploads and Metadata*

* **File Import**: Uploaded images appear in the sidebar and are added to the database. Success threshold: 100% of valid files import successfully.
* **Metadata Editing**: Updates to line count, script type, edge flags, circle, scale, and custom fields persist in SQLite. Success threshold: 100% of edits saved in test cases.

*ML Pipeline*

* **Segmentation and Metadata Enrichment**: Running the pipeline updates segmentation coordinates, line counts, script types, edges, circles, and scale data where enabled. Success threshold: >=90% of fragments in a test batch are processed without pipeline errors.

==== 2. Non-Functional Verification Criteria

*Performance*

* **UI Responsiveness**: Sidebar paging and filtering remain usable on large datasets. Success threshold: no UI freezes >1s during typical navigation.

*Reliability*

* **Local Data Integrity**: Database writes are atomic and survive application restart. Success threshold: no data loss after restart in test scenarios.

==== 3. Academic Research Acceptance Criteria

*Research Workflow Validation*

* **Scholarly Utility**: Researchers can complete a basic reconstruction workflow (search, place, save, restore, edit metadata) without assistance. Success threshold: >=80% task completion in user testing.

==== 4. Testing and Validation Framework

*Testing Approach*: Unit tests for ML processors, integration tests for database updates, and manual UI walkthroughs for canvas and project flows.

*Acceptance Decision*: Platform is ready for use when the desktop app workflows function reliably, uploads and metadata edits persist, and the ML pipeline can enrich a sample dataset without critical failures.
