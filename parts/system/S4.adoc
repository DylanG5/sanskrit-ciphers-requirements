In this section we take the use cases from section G.5 and give a detailed flow to them here.

=== (S.4.1) Start a Reconstruction Session

**Normal flow**

* The scholar launches the desktop application.
* The home screen lists recent projects and a fragment search bar.
* The scholar creates a new project or selects an existing project.
* The system opens the canvas workspace.

**Alternate flow**

* The scholar enters a fragment ID in the search bar and submits.
* The system opens the canvas and auto-places the first matching fragment.

**Error flow**

* If the local database is unavailable, the system shows an error and disables fragment browsing.
* If a project fails to load, the system returns to the home screen with an error message.


[#s4-2,reftext=S.4.2]
=== (S.4.2) Browse, Filter, and Place Fragments

**Normal flow**

* The scholar opens the filter panel and sets criteria (line count, script type, edge flags, circle, custom fields).
* The system updates the sidebar list with matching fragments.
* The scholar drags fragments from the sidebar onto the canvas.
* The scholar moves, resizes, rotates, and locks fragments as needed.

**Alternate flow**

* The scholar toggles the grid for scale alignment.
* The scholar toggles segmented/original view for a selected fragment.

**Error flow**

* If an image fails to load, the system displays an error placeholder and keeps the remaining fragments available.


[#s4-3,reftext=S.4.3]
=== (S.4.3) Save and Restore a Project

**Normal flow**

* The scholar edits the canvas.
* The system auto-saves after changes.
* The scholar optionally clicks "Save" to force a save.
* The scholar later selects the project from the home screen.
* The system restores the canvas layout.

**Alternate flow**

* The scholar renames or deletes a project from the home screen.

**Error flow**

* If saving fails, the system marks the project as unsaved and retains the current canvas state until the next successful save.


[#s4-4,reftext=S.4.4]
=== (S.4.4) View and Edit Metadata

**Normal flow**

* The scholar double-clicks a fragment on the canvas (or selects one from the list).
* The system opens the metadata panel with current values.
* The scholar edits fields such as line count, script type, edge flags, circle, scale, and custom fields.
* The system validates and saves the updates to the database.

**Alternate flow**

* If the fragment was resized on the canvas, the scholar uses "Set Scale from Resize" to update pixels-per-unit.

**Error flow**

* If validation fails or the database update fails, the system displays an error and preserves the previous value.


[#s4-5,reftext=S.4.5]
=== (S.4.5) Upload New Fragments

**Normal flow**

* The scholar opens the upload dialog.
* The scholar selects one or more image files (JPG, PNG).
* The system copies files into the local data directory and adds database records.
* The new fragments appear in the sidebar list.

**Alternate flow**

* The scholar uploads multiple files in a single batch.

**Error flow**

* If a file fails to import, the system reports the failure and continues with the remaining files.
