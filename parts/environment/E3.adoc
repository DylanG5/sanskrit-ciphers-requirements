=== (E.3) Constraints

ifdef::env-draft[]
TIP: _Obligations and limits imposed on the project and system by the environment. This chapter defines non-negotiable restrictions coming from the environment (business rules, physical laws, engineering decisions), which the development will have to take into account._  <<BM22>>
endif::[]


The system will be bounded by several constraints which will restrict its design and implementation.

*Data Privacy Regulations*: The system must comply with relevant data privacy laws. This includes ensuring that fragment images and metadata stored locally are protected and handled appropriately.

*Performance Requirements*: The system must keep the desktop UI responsive when browsing and manipulating large fragment sets.

*Security Standards*: The system must protect local data from accidental loss or corruption, including safe database writes and backups where applicable.
