==== Core System Functions

The Sanskrit Manuscript Fragment Reconstruction Platform provides a local desktop interface for Buddhist Studies scholars to search, filter, and reconstruct ancient texts using a local SQLite dataset. The system operates through three primary functional domains:

==== 1. Fragment Processing and Metadata Enrichment

*Image Segmentation and Feature Extraction*
* Segment fragment images to isolate manuscript content
* Extract contour-based metadata (edge flags, edge piece)
* Detect circles and estimate scale from rulers where present

*Metadata Extraction*
* Predict script type labels
* Predict line count and line detections
* Store metadata in a searchable database

==== 2. Search, Filtering, and Edge Matching

*Multi-Criteria Filtering Interface*
* Allow researchers to enter specific fragment IDs for targeted searches
* Filter fragments by script type, line count, edge flags, circles, and custom fields
* Combine multiple filter criteria to narrow search results

*Edge Matching*
* Compare tear-edge descriptors to produce candidate matches

==== 3. Interactive Reconstruction Workspace

*Canvas-Based Assembly Interface*
* Provide drag-and-drop canvas for arranging fragment images
* Enable manipulation of fragment positions, size, and rotation
* Support grid alignment and segmented/original views

*Session Management*
* Save and restore reconstruction work sessions
* Support multiple reconstruction projects with auto-save

==== Integration and Data Management

*Local Data Store*: Fragment images and metadata are stored locally in a SQLite database.
*User Image Submission*: Researchers can upload new fragment images for inclusion in the dataset.
*Custom Metadata*: Users can define and edit custom metadata fields for filtering and analysis.

==== Workflow Integration

The system supports a streamlined workflow: researchers search or filter fragments → drag fragments to the canvas to test arrangements → edit metadata as needed → save reconstruction sessions for later → run the ML pipeline to enrich metadata on new or unprocessed fragments.
