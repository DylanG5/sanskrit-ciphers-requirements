==== System Overview

The system will support scholars of Buddhist textual history by providing tools that automate and accelerate the process of reconstructing fragmented manuscripts. The following main usage scenarios capture the essential ways in which users will interact with the system:

==== Search, Filter, and Place Fragments

* A scholar searches by fragment ID or applies filters (line count, script type, edge flags, circles, custom fields).
* The system returns matching fragments in the sidebar.
* The scholar drags fragments onto the canvas and arranges them.

==== Edge Matching and Reconstruction

* The scholar selects a fragment and requests edge-based candidate matches.
* The system produces candidate matches based on edge descriptors.
* The scholar inspects candidates and arranges fragments on the canvas.

==== Metadata Viewing and Editing

* The scholar opens a fragment’s metadata panel.
* The system displays current metadata and ML-generated values.
* The scholar edits fields (line count, script type, edge flags, circle, scale, custom fields) as needed.

==== Upload New Fragments

* The scholar uploads one or more fragment images.
* The system adds them to the local data store for browsing and processing.

==== Run the ML Pipeline

* The scholar or operator runs the local ML pipeline.
* The system updates segmentation and metadata fields for fragments.
